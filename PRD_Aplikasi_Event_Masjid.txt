PRODUCT REQUIREMENT DOCUMENT (PRD)
Sistem Pendaftaran Event Masjid (Mosque Event Registration System)
1. Ringkasan Proyek (Project Overview)
Aplikasi ini adalah platform berbasis web responsif yang dirancang untuk mendata jemaah yang ingin mendaftar ke
berbagai event yang diadakan oleh masjid. Fokus utama dari sistem ini adalah memberikan kemudahan alur
pendaftaran bagi jemaah mandiri, proteksi dari robot/spam, serta menyediakan panel kontrol yang fleksibel bagi
Admin Masjid untuk mengelola data peserta dan memperbarui informasi event secara dinamis.
Tujuan Utama
• Mempermudah jemaah melakukan pendaftaran event secara mandiri tanpa memerlukan bantuan fisik panitia.
• Mencegah terjadinya duplikasi data pendaftar dengan validasi berbasis nomor unik WhatsApp.
• Memberikan fleksibilitas bagi pengurus/admin masjid dalam mengubah detail event tanpa perlu mengubah
struktur kode aplikasi.
2. Peran Pengguna (User Roles)
Peran Pengguna 	Deskripsi & Hak Akses
Pendaftar (Jemaah) 	Masyarakat umum atau jemaah masjid yang mengakses halaman publik untuk
melakukan registrasi mandiri. Tidak membutuhkan akun untuk login.
Admin Masjid 	Pengurus takmir atau panitia event yang memiliki hak penuh untuk mengakses
Dashboard Admin, melakukan manajemen data, serta memperbarui nama event.

5. Struktur Data (Data Structure)
Untuk mendukung perancangan sistem, direkomendasikan struktur minimal dua tabel berikut:
Tabel: `events` (Konfigurasi Event)
Menyimpan data nama event dinamis yang dikendalikan dari dashboard admin.
`id` (INT, Primary Key, Auto Increment)
`event_name` (VARCHAR, 255) - Nama event aktif
`updated_at` (TIMESTAMP) - Penanda waktu pembaruan nama event
Tabel: `participants` (Data Pendaftar)
Menyimpan riwayat entri jemaah masjid.
`id` (INT, Primary Key, Auto Increment)
`participant_number` (VARCHAR, 50, Unique) - Nomor peserta ter-generate otomatis (e.g., MSG-001)
`name` (VARCHAR, 150) - Nama lengkap pendaftar
`whatsapp_number` (VARCHAR, 20, Unique Index) - Nomor kontak WA utama
`gender` (ENUM('L', 'P')) - Representasi jenis kelamin
`created_at` (TIMESTAMP) - Tanggal dan waktu join pendaftaran
